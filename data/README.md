# Data Setup

This directory holds the benchmark data. **Do not commit PHI or large data files** — the `.gitignore` is configured to exclude them.

## Required Files

### 1. Ground Truth — `ground_truth.csv`

The question/answer/FHIR-ID evaluation dataset from [FHIR-AgentBench](https://github.com/glee4810/FHIR-AgentBench).

**Quick path:** run `bash scripts/setup_data.sh` — it will shallow-clone the repo and copy the CSV here.

**Manual path:** download `final_dataset/questions_answers_sql_fhir.csv` from the repo and place it at `data/ground_truth.csv`.

Key columns:

| Column | Description |
|---|---|
| `question_id` | Unique identifier |
| `question` | Natural-language clinical question |
| `true_answer` | Ground-truth answer |
| `true_fhir_ids` | Dict mapping FHIR resource types → list of resource IDs (e.g., `{"Patient": ["Patient/12345"], "Condition": ["Condition/67890"]}`) |

### 2. FHIR Records — `fhir_records/`

NDJSON files from the [MIMIC-IV Clinical Database Demo on FHIR](https://physionet.org/content/mimic-iv-fhir-demo/2.1.0/) (PhysioNet, requires credentialed access).

Run `bash scripts/setup_data.sh` to download, extract, and build the local document store. The setup script automatically strips the `Mimic` prefix from the PhysioNet file names so they align with standard FHIR resource types.

After setup, `data/fhir_records/` will contain files like:

```
data/fhir_records/
├── Patient.ndjson
├── Condition.ndjson
├── ConditionED.ndjson
├── Encounter.ndjson
├── EncounterED.ndjson
├── EncounterICU.ndjson
├── MedicationRequest.ndjson
├── MedicationAdministration.ndjson
├── MedicationDispense.ndjson
├── ObservationChartevents.ndjson
├── ObservationLabevents.ndjson
├── Procedure.ndjson
├── ProcedureED.ndjson
├── ProcedureICU.ndjson
└── ...  (30 files total)
```

If you prefer to download manually, grab the `.ndjson.gz` files from PhysioNet, decompress them, strip the `Mimic` prefix from each filename, and place them in `data/fhir_records/`.

### 3. Document Store — `fhir_store.db`

Auto-generated by the setup script. A SQLite key-value store mapping FHIR resource IDs to raw JSON. If you need to rebuild it, delete this file and re-run setup.
